RHEL各版本的XManager4远程配置

Red Hat 7,Red Hat 8,Red Hat 9,Red Hat 3.0 AS 下配置 XDMCP 连接
1. XDM Configuration
1.将系统的运行级别设置为5，具体如下：
打开/etc/inittab。修改参数 id：5：initdefault;
2. 激活 XDMCP
为 GDM:
打开/etc/X11/gdm/gdm.conf,然后在打开的编辑中为[xdmcp]设置一个参数如下：
[xdmcp]
Enable=1
For KDM: #RHEL默认是GDM，没有安装KDM的下面的路径是找不到的，所以不用去配置这个，这里只是给个概念。以下也是一样。
打开/etc/X11/xdm/xdm-config并做以下修改：
DisplayManager.requestPort: 0 ==>
!DisplayManager.requestPort: 0
打开/etc/X11/xdm/Xaccess，并做以下修改：
#* #any host can get a login window ==>
* #any host can get a login window
在[xdmcp]下添加Enable=true。格式如下：[xdmcp]
Enable=true
2. 防火墙配置
如果我们的Linux机器配置有防火墙，为防止防火墙将通过177端口（即xdmcp服务）的数据过虑。
#iptables -A INPUT -p udp -s 0/0 -d 0/0 --dport 177 -j ACCEPT。
若没有防火墙则可跳过这一项
3. 重新启动远程主机，并启动 XManager
# init 3; init 5
 

Red Hat 5 EL 下配置 XDMCP
1. XDM Configuration
1.将系统的运行级别设置为5，具体如下：
打开/etc/inittab。修改参数 id：5：initdefault;
2. 激活 XDMCP
For GDM:
打开/etc/X11/gdm/gdm.conf,然后在打开的编辑中为[xdmcp]设置一个参数下：
[xdmcp]
Enable=1
For KDM:
打开/usr/share/config/kdm/Xaccess,将编辑文本中的初始化参数换成（remove掉一个#）：
#* #any host can get a login window ==>
* #any host can get a login window
打开/usr/share/config/kdm/kdmrc，在[xdmcp]下添加Enable=true。格式如下：
[xdmcp]
Enable=true
2. 防火墙配置
如果我们的Linux机器配置有防火墙，为防止防火墙将通过177端口（即xdmcp服务）的数据过虑。
#iptables -A INPUT -p udp -s 0/0 -d 0/0 --dport 177 -j ACCEPT。
若没有防火墙则可跳过这一项
3. 重启远程主机，并启动 XManager
# init 3; init 5
