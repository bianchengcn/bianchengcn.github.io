数据总线、地址总线、字、字长、字节和位的概念及关系

详细梳理数据总线、地址总线、字、字长、字节和位的概念及关系。 欢迎指正、讨论、关注、分享~ 一、计算机总线 总线是CPU、内存、输出/输出设备传递信息的公用通道。在计算机系统中，主机的各个部件通过总线相连接，外部设备通过相应的接口电路与总线相连接。按照计算机所传输的信息种类，计算机总线分为：控制总线CB、数据总线DB和地址总线AB。 1. 控制总线 通过传递控制信号和时序信号（如读/写信号、中断响应信号等）对外部器件进行控制。 2. 数据总线 用于传送数据信息，是双向三态形式的总线。 双向： ①将CPU的数据传送到存储器（RAM、ROM）或I/O接口等其他部件 ②将其他部件的数据传送到CPU 三态： 高水平（1）、低水平（0）、高组态（Z） 数据总线的宽度一般指CPU中运算器与存储器之间进行互连的内部总线二进制位数，即CPU一次读取的二进制位数（内部总线根数）。数据总线宽度决定了就是计算机一次所能传递的二进制位数（数据量），决定了在主存储器和CPU之间的数据交换效率 。数据总线宽度是决定计算机性能的一个重要指标。 数据总线宽度说明字长，数据总线宽度为n ，字长为n位(bit) 3. 地址总线 用于传送地址信息，由于地址只能从CPU传向外部存储器或I/O端口，所以地址总线是单向三态的。 地址总线的位数，即地址总线宽度，即地址总线根数 地址总线的位数决定了CPU可直接寻址的内存空间大小，决定了CPU能直接访问的内存单元的个数。 内存单元=（字节）存储单元 内存按字节（byte）编址，每个地址指向的存储单元可以保存一个字节的数据。即每个存储单元内存空间大小为1Byte 地址总线宽度为n ，则CPU可直接寻址空间大小为2^n字节，CPU可直接访问的内存单元个数为2^n 二、字和字长 1. 字 在计算机中，字是其用来一次性处理事务的一个固定长度的位组。 存储单元： 存放一个机器字的存储单元，通常称为字存储单元，其单元地址称为字地址。 存放一个字节的存储单元，称为字节存储单元，其单元地址称为字节地址。 计算机中可以编址的最小单元是字存储单元，则该计算机为按字寻址的计算机。 计算机可编址的最小单位是字节，则该计算机为按字节寻址的计算机。 例如：一个32位计算机，一个字=32位=4字节 2. 字长 字长，一个字的二进制位数（bit）， 指的是CPU一次能并行处理的二进制位数。 字长直接反映了一台计算机的计算精度，在其他指标相同时，字长越大计算机的处理数据的速度就越快。 例如：一个32位计算机，其字长为32位 三、字节和位 1字节（Byte/B） = 8 位（bit/b）

都说计算机今年炸了，究竟炸到什么程度呢?

近期，最大的计算机领域新闻莫过于Intel CPU严重漏洞曝光。该漏洞被称为“Meltdown”和“Spectre”，几乎涵盖了所有使用Intel芯片的计算机，包括PC、笔记本电脑、服务器和移动设备。 Meltdown漏洞的主要风险是黑客可以利用此漏洞访问操作系统的核心内存，从而获得操作系统中存储的所有敏感数据，如密码、私钥、加密密钥等。而Spectre则可以允许攻击者通过危险的JavaScript代码从一个网站窃取另一个网站的信息。 此次漏洞严重影响了全球各个领域的计算机安全问题，包括银行、信用卡公司、云计算、医疗保健等。 不过，幸运的是目前市场上已经有多家厂商发布了补丁程序，其中包括Microsoft、Apple、Amazon、Google、Linux等。而且，Intel公司已承诺将为这些公司提供技术支持和更改设计以解决这些漏洞。 尽管这一事件引起了一些担忧，但人们依然相信计算机仍然是目前社会运转的基石之一，并且相信在科技的帮助下，我们能够应对和解决这些问题。 此次漏洞的曝光也进一步提醒人们在日常使用计算机时的安全意识。作为普通用户，应当避免不必要的访问来路不明的网站和程序，保护好自己的网络账号和密码，不轻易输入个人身份信息等。同时，应该及时更新操作系统和安装的软件以及防病毒软件以保证计算机的安全。 而对于企业、政府机构等，更需要加强网络安全防护措施，在网络安全管理上加强控制和监督，并定期组织员工进行相关安全教育和培训，以切实保障组织运转的安全性。 最后，我们要注意的是，计算机技术的发展速度是非常快的，而漏洞的发现以及修复难以完全规避，因此保持安全意识和及时更新软件与系统是长久以来保护计算机安全的有效手段。 同时，对于漏洞的修复也需要采取多种手段。除了开发者发布补丁外，用户和企业也可以采取以下措施： 限制软件的使用权限。比如，操作系统和应用程序可以设置使用权限，避免非管理员账户对系统敏感信息和配置进行修改。 防火墙和入侵检测系统的帮助。防火墙和入侵检测系统可以阻止攻击者从计算机网络访问电脑系统，并对恶意流量进行检测和阻止。 安全软件的使用。安全软件能够通过不断更新特征库，以便及时发现和阻止病毒、恶意软件，保障计算机及相关数据的安全。 及时备份数据。在备份时要注意备份数据的完整性和恢复数据的可靠性。 总之，计算机安全是一个重要问题，需要个人和组织共同关心和维护。只有在大家的共同努力下，才能进一步提升计算机的安全水平。 此外，随着技术的不断进步，安全风险也在不断增加。如今的云计算、物联网、人工智能等新兴技术，也给注重安全的企业和个人带来更多挑战。 针对这些新兴技术带来的威胁，我们需要不断学习和探究相关安全知识和解决方案，同时也需要注重网络安全法规和标准。如欧盟通用数据保护条例（GDPR）、全球数据保护条例（GPDR）等，都为互联网治理和信息安全提供了有力的支撑和引导。 综上所述，计算机安全是一个长期而艰巨的任务，需要我们不断地关注和投入精力。只有透过漏洞修复、技术完善和监管加强等多重手段，才能实现计算机及数字化世界的安全有序运转。 此外，提高个人安全意识也是保障计算机安全的重要环节。许多网络攻击都是通过利用用户的疏忽或无知来实现的。因此，我们需要了解一些基本的安全知识，比如设置复杂的密码，不轻易泄露个人信息，不随意打开邮件附件或下载来路不明的软件等。 企业和机构应该建立健全的安全管理制度和体系，包括完善的安全防控方案、网络安全培训和演练、及时的安全漏洞修复等。同时，还需要引进专业的安全产品和服务，如入侵检测与防御系统、数据加密技术、安全服务外包等，为企业和机构提供更可靠的保障。 总之，保障计算机安全是一个千头万绪的工程，需要广泛的意识宣传、政府、企业和组织的合作以及人类智慧和努力。只有全社会共同参与才能真正实现数字世界的安全发展。

Manjaro的EFI系统分区迁移（通用Linux）

EFI系统分区迁移 前言 我是用的是Manjaro，由于当时/home目录的分区太小了，而Linux上扩展分区只能连续的分区中扩展，我的efi堵在了中间，所以就想把他迁移到另一个分区，顺利扩展/home目录分区。 步骤 以下是在同一个硬盘中迁移中的EFI系统分区的详细步骤： 打开终端，并使用以下命令卸载要迁移的EFI系统分区： sudo umount /boot/efi 使用 lsblk 命令或其他分区工具查看硬盘的分区布局，确定要迁移的EFI系统分区及其设备文件名和UUID。 使用以下命令创建一个与要迁移的EFI系统分区大小相同的新分区： sudo fdisk /dev/sda n p <Enter> <Enter> <Enter> t <Enter> 1c w 其中，/dev/sda 是硬盘的设备文件名，n 命令创建新分区，p 命令指定分区类型为主分区，t 命令更改分区类型，1 是要更改的分区编号，c 是EFI系统分区的类型代码，w 命令将更改写入硬盘。 格式化新分区为FAT32文件系统： sudo mkfs.vfat -F32 /dev/sdaY 其中，/dev/sdaY 是新分区的设备文件名，Y 是新分区的编号。 挂载原始EFI系统分区和新分区： sudo mount /dev/sdaX /mnt sudo mount /dev/sdaY /mnt2 其中，/dev/sdaX 是要迁移的EFI系统分区的设备文件名，/dev/sdaY 是新分区的设备文件名，/mnt 和 /mnt2 是挂载点。 使用以下命令将原始EFI系统分区的内容复制到新分区： sudo cp -a /mnt/. /mnt2/ 其中，/mnt/ 是原始EFI系统分区的挂载点，/mnt2/ 是新分区的挂载点。 更新新分区的 /etc/fstab 文件，并将新分区的UUID添加到其中： sudo blkid /dev/sdaY 其中，/dev/sdaY 是新分区的设备文件名，它将返回新分区的UUID。

一个服务包含一个或多个特性，这些特性是逻辑上相关的集合体。Characteristic 一个特性至少包含2个属性：一个属性用于声明(又叫描述符)，一个属性用于存放特性的值。 存放特性值的属性就是真正传输的数据，声明属性用来确定该特性是否可读写、是否可以发起通知（类似于向主机发起中断）等信息。 比如，LED状态就是一个特性，该特性可读写。Properties 特性的操作类型有：写、没有回应的写、读、通知、指示（有回应的通知） 通过字面意思就可以理解，比如LED特性，需要写和读性质。如果设备有事件需要上报，比如按键等，就需要通知或者指示性质。通知就是上报完就没事了，指示的话还需要主机给个响应。Descriptors 描述符就是用于声明的属性。 有一个特别的描述符值得特别地提起：客户端特性配置描述符(Client Characteristic
