centos6图形化启动失败，一直卡在进度条界面

虚拟机有一段时间没用，开机发现linux系统卡在开机画面了，不能正常开机

开始分析排查问题

1.du启动虚拟机，按任意键进入GUN界面

按e进入

 

移到第二项，按e进行编辑

把rhgb quiet删掉，回车，按b重新引导系统。

结果执行到一半又不动了，整个人都不好了

2. 直接快捷键 ctrl+alt+f2进入命令行模式，使用root账号登录，修改一下启动级别，不使用图形化界面了

使用vim /etc/inittab 命令

# Default runlevel. The runlevels used are:

#   0 - halt (Do NOT set initdefault to this)

#   1 - Single user mode

#   2 - Multiuser, without NFS (The same as 3, if you do not have networking)

#   3 - Full multiuser mode

#   4 - unused

#   5 - X11

#   6 - reboot (Do NOT set initdefault to this)

#

id:5:initdefault:#把最底行的5改为3，即默认进入命令行操作界面

：wq保存

结果竟然无法保存！！！

提示 system error （file system full）

文件系统满了，什么鬼，除了部署环境，里边什么都没有

3.继续查找原因

在根路径下使用df -h 查看磁盘的使用情况，结果一看使用竟然达到100%

使用du -sh *命令，查看每个文件夹的大小

发现/var/log/maillog日志文件竟然达到43g

使用rm /var/log/maillog -rf 删除日志文件

重启虚拟机，ok，正常启动了

至于为什么maillog日志文件这么大，还没有找到原因，继续观察吧。
